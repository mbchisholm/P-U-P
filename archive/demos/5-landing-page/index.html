Curriculum

Five parts. One complete picture.

Each part builds on the last. By the end you'll understand the full journey from a blank terminal to a live server on the internet.

Part 01

The File System & Shell

Open a terminal for the first time and learn to navigate, create, move, and delete files without touching a mouse. This is how every developer — and every server — works.

terminal bash filesystem
Part 02

The Lay of the Land

Six quick demos — bash pipelines, static websites, Python scripts, Node projects, a designed landing page, and generative art. See what different types of projects look like before you build one.

python node html/css p5.js
Part 03

Build the Real Thing

Write a Flask server that streams live flight radar and ISS position to a 3D globe in the browser — with a shared message board everyone in the room can post to in real time.

flask websockets APIs globe.gl
Part 04

GitHub

Commit your project, push it to GitHub, and understand why version control exists. This also sets up your deployment in Part 5 — the Pi will clone directly from your repo.

git github version control
Part 05

Networking & Deployment

Flash a Raspberry Pi Zero, SSH into it from your laptop, and deploy your app to it. When it's running on the Pi, anyone in the room can visit it from their phone. That's what "hosting" means.

ssh raspberry pi nmap networking
What You'll Build

A live flight radar.
Running on a $15 computer.

Real planes. Real position data. A 3D globe. Shared in real time with everyone in the room — served from a chip the size of a stick of gum.
